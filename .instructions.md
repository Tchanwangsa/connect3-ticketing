## Core Principles

### 1. Zero Code Duplication

**❌ DO NOT duplicate code, utilities, types, or hooks.** Always extract reusable logic into:

- **Utils**: `lib/utils/` (pure functions)
- **Hooks**: `lib/hooks/` (React hooks for component logic)
- **Components**: `components/shared/` (reusable UI components)
- **Types**: `lib/types/` (shared TypeScript interfaces)

**Action**: Before implementing a new utility/hook/type, search the codebase for existing implementations.

### 2. Always Use Existing Utilities

When working with events, timezones, images, auth, etc., use the existing utilities:

- **Timezone conversion**: `lib/utils/timezone.ts`
- **Image handling**: `lib/utils/cropImage.ts`, `lib/utils/uploadEventImage.ts`
- **Club admin checks**: `lib/auth/clubAdmin.ts`
- **Auto-save**: `lib/hooks/useAutoSave.ts`
- **Real-time updates**: `lib/hooks/useEventRealtime.ts`

### 3. Centralized Type Definitions

**❌ DO NOT define the same type in multiple files.** Define types ONCE in `lib/types/` and import everywhere.

**Action**: Check `lib/types/events.ts` and `lib/types/ticketing.ts` before creating new types.

### 4. Component Structure

- Break large components into smaller, focused sub-components (max 200 lines per component)
- Extract component logic into custom hooks in `lib/hooks/`
- Import all props as TypeScript interfaces
- Use composition over prop drilling

### 5. API Client Functions

**❌ DO NOT make raw fetch calls.** Always use API client functions from `lib/api/`.

```typescript
// ✅ GOOD
import { fetchEvent } from "@/lib/api/fetchEvent";
const event = await fetchEvent(eventId);

// ❌ BAD
const res = await fetch(`/api/events/${eventId}`);
```

## Critical Constraints (READ CAREFULLY)

### Timezone Handling for Events

**CRITICAL**: Event timestamps stored in the database must be in UTC.

- Convert user's local time to UTC using `convertToUTC()` before storing
- **DO NOT** split ISO strings directly - this shows UTC wall-clock values
- Always include the event timezone in the database record
- When retrieving events, convert back to the event timezone using `convertToEventTimezone()`

**Example**:

```typescript
import { convertToUTC } from "@/lib/utils/timezone";

const eventData = {
  start_time: convertToUTC(new Date(body.startTime), body.timezone),
  end_time: convertToUTC(new Date(body.endTime), body.timezone),
  timezone: body.timezone,
};
```

### Club-Scoped Operations

**CRITICAL**: For CREATE/UPDATE/DELETE operations on club resources:

1. Resolve the acting profile from `club_id`, not the authenticated user ID
2. Verify `club_admin` access explicitly
3. **DO NOT** silently fall back to user profile

**Example**:

```typescript
const isAdmin = await isClubAdmin(user.id, clubId);
if (!isAdmin) {
  return NextResponse.json({ error: "Forbidden" }, { status: 403 });
}
```

### Instagram Post Imports (Club Admins)

Query `instagram_club_fetches` by the selected club **profile_id**, not the authenticated user ID.

See [docs/api/PATTERNS.md](./docs/api/PATTERNS.md#3-instagram-club-post-imports) for details.

## Before Implementing Any Feature

### Checklist

- [ ] Read relevant documentation in `docs/`
- [ ] Search for existing utilities/hooks/types that do the same thing
- [ ] Check if similar components already exist
- [ ] Verify timezone handling (if events are involved)
- [ ] Verify club-scoped access control (if clubs are involved)
- [ ] Create custom hooks for component logic (don't inline large logic)
- [ ] Define types in `lib/types/` if they're shared
- [ ] Use existing Supabase clients (client/server/admin)
- [ ] Update `docs/DOCUMENTATION.md` and related docs after completing the feature

## When Refactoring Code

### Action Items

1. Extract reusable logic into `lib/hooks/` or `lib/utils/`
2. Move reusable components to `components/shared/`
3. Centralize duplicated types in `lib/types/`
4. Update documentation if you discover new patterns
5. Update this file if you find critical constraints that should be documented

### Anti-Patterns to Remove

- Raw fetch calls → Use `lib/api/` functions
- Duplicate types → Centralize in `lib/types/`
- Inline timezone conversion → Use `lib/utils/timezone.ts`
- Club membership logic in every file → Use `lib/auth/clubAdmin.ts`
- Large components with mixed concerns → Break into smaller components + hooks
